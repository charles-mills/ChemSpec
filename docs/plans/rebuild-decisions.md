# Dynamic rebuild decisions

- 2026-07-16 DYN-104: the pre-rebuild compact-plan caller and both legacy repair prompts were removed after the typed outcome compiler replaced the final consumer. No live path can request `.chems`, working catalogues, evidence packets, structures, mappings, or operations in the factual-claim call.
- 2026-07-16 DYN-105: extended the existing domain `ResolvedSpecies` contract instead of creating a second competing resolved-species type; external chemistry machinery remains behind future adapters, while this slice implements deterministic reviewed/cache resolution for the current simple species library.
- 2026-07-16 DYN-105: catalogue structures do not encode a generally authoritative phase, so migrated reviewed identities use the new explicit `Unknown` phase until a request or identity source supplies a checked qualifier; no phase is guessed from representation.
- 2026-07-16 DYN-106: a claimed product absent from the checked identity registry may produce only a neutral formula-only static card; it receives no fabricated graph and cannot enter mechanism escalation until an identity adapter supplies a validated structure and charge.
- 2026-07-16 DYN-105: when the legacy element-composer input is a one-element formula with multiple catalogue aliases for the same reviewed neutral metallic composition, the compatibility resolver selects the stable lowest identity; all chemically distinct multi-identity results remain explicit ambiguity until DYN-109 supplies richer composer qualifiers.
- 2026-07-16 DYN-108: the escalation wire contract is a direct closed JSON schema rather than a string-wrapped ledger. Codex receives only locally labelled structures and coefficients; every returned label is checked before the proposed rule enters the existing kernel.
- 2026-07-16 DYN-108: every checked declaration attempts the existing reviewed-family registry before mechanism escalation, including Fast `ModelAsserted` claims. A successful kernel-validated family match upgrades the displayed factual tier to `Reviewed`; only a real local miss may call the mechanism provider.
- 2026-07-16 DYN-108: mechanism escalation uses low reasoning, the default service tier, no live search, a 120-second per-attempt bound, and at most two operation-level repairs. Exhaustion preserves the validated static outcome and exposes retry instead of treating animation failure as chemistry failure.
- 2026-07-16 DYN-109: the application commits an exactly balanced static result before launching presentation enrichment. Reviewed-family and escalated animations share playback, while static outcomes expose no frames and explicitly distinguish pending, unavailable, and retryable mechanism states.
- 2026-07-16 DYN-109 (superseded 2026-07-17): the builder now exposes one implicit low-latency claim path and no persisted mode preference. Generation IDs still gate claim, verification, and presentation completions; verification upgrades the badge without replacing an existing playback artefact.
- 2026-07-16 DYN-110: cache v3 stores only an untrusted claim, digest-bound evidence snapshot, and a revalidation recipe for reviewed or escalated presentation. It never serializes a trusted capability; every offline load recompiles balance, restores evidence coverage, and crosses family/kernel validation again.
- 2026-07-16 DYN-110: `ReactionBuildRequest` carries an optional selected-context string separately from resolved reactant identity. Cache binding normalizes and hashes that context with the stable identities, while the current element composer deliberately supplies no context until it has a real user control.
- 2026-07-16 DYN-111 (updated 2026-07-17): release invocation unconditionally uses low reasoning and the default service tier. Initial claims, source verification, and mechanism escalation have 30, 90, and 120 second bounds respectively; normalized progress deliberately drops model text and reasoning.
- 2026-07-18 DYN-111 runtime isolation: native claim and presentation pipelines run through Tokio's blocking pool instead of occupying an Iced async runtime worker. Completion still returns through the existing typed generation-scoped messages; the provider's atomic flag remains the cooperative cancellation authority because a started blocking task cannot be force-aborted, and app teardown signals that flag before runtime shutdown. The wasm build remains direct because it forces Local Mode and cannot launch the native provider.
- 2026-07-18 DYN-111 failure contract: agent failures expose a closed `AgentErrorKind` for cancellation, timeout, capability, provider, output, cache, identity, request, compilation, kernel, and internal boundaries. Human-readable context is diagnostic only; concrete I/O, JSON, domain, and kernel errors remain available through `Error::source`, and the application retains the typed failure through its generation-scoped worker completion and failed-build state.

- 2026-07-16 DYN-108 (parity repair): reviewed aliases of one species can differ only in internal atom labels (for example `SodiumChloride` vs `NaFixedCation1Chloride`), which defeated label-sensitive graph equality during family matching. `ValidatedCatalogueBundle::structures_isomorphic` now proves exact label-independent equivalence with the bounded automorphism search machinery; constitutional isomers remain distinct, and a hit work limit fails closed to the resolver's own pick.
