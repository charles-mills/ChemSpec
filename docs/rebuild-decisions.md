# Dynamic rebuild decisions

- 2026-07-16 DYN-104: the pre-rebuild compact-plan caller and both legacy repair prompts were removed after the typed outcome compiler replaced the final consumer. No live path can request `.chems`, working catalogues, evidence packets, structures, mappings, or operations in the factual-claim call.
- 2026-07-16 DYN-105: extended the existing domain `ResolvedSpecies` contract instead of creating a second competing resolved-species type; external chemistry machinery remains behind future adapters, while this slice implements deterministic reviewed/cache resolution for the current simple species library.
- 2026-07-16 DYN-105: catalogue structures do not encode a generally authoritative phase, so migrated reviewed identities use the new explicit `Unknown` phase until a request or identity source supplies a checked qualifier; no phase is guessed from representation.
- 2026-07-16 DYN-106: a claimed product absent from the checked identity registry may produce only a neutral formula-only static card; it receives no fabricated graph and cannot enter mechanism escalation until an identity adapter supplies a validated structure and charge.
- 2026-07-16 DYN-105: when the legacy element-composer input is a one-element formula with multiple catalogue aliases for the same reviewed neutral metallic composition, the compatibility resolver selects the stable lowest identity; all chemically distinct multi-identity results remain explicit ambiguity until DYN-109 supplies richer composer qualifiers.
- 2026-07-16 DYN-108: the escalation wire contract is a direct closed JSON schema rather than a string-wrapped ledger. Codex receives only locally labelled structures and coefficients; every returned label is checked before the proposed rule enters the existing kernel.
- 2026-07-16 DYN-108: every checked declaration attempts the existing reviewed-family registry before mechanism escalation, including Fast `ModelAsserted` claims. A successful kernel-validated family match upgrades the displayed factual tier to `Reviewed`; only a real local miss may call the mechanism provider.
- 2026-07-16 DYN-108: mechanism escalation uses low reasoning, the default service tier, no live search, a 120-second per-attempt bound, and at most two operation-level repairs. Exhaustion preserves the validated static outcome and exposes retry instead of treating animation failure as chemistry failure.
- 2026-07-16 DYN-109: the application commits an exactly balanced static result before launching presentation enrichment. Reviewed-family and escalated animations share playback, while static outcomes expose no frames and explicitly distinguish pending, unavailable, and retryable mechanism states.
- 2026-07-16 DYN-109: Fast/Researcher is a persisted claim-mode preference, not the Codex Fast service tier. Generation IDs gate claim, verification, and presentation completions; verification upgrades the badge without replacing an existing playback artefact.
- 2026-07-16 DYN-110: cache v3 stores only an untrusted claim, digest-bound evidence snapshot, and a revalidation recipe for reviewed or escalated presentation. It never serializes a trusted capability; every offline load recompiles balance, restores evidence coverage, and crosses family/kernel validation again.
- 2026-07-16 DYN-110: `ReactionBuildRequest` carries an optional selected-context string separately from resolved reactant identity. Cache binding normalizes and hashes that context with the stable identities, while the current element composer deliberately supplies no context until it has a real user control.
- 2026-07-16 DYN-111: release invocation unconditionally uses low reasoning and the default service tier. Fast claims, Researcher claims, and mechanism escalation have 30, 90, and 120 second bounds respectively; normalized progress deliberately drops model text and reasoning.
- 2026-07-16 DYN-111: process-local preflight caching retains only successful Codex version/capability probes keyed by executable path; `codex login status` still runs on every preflight so cached capabilities never imply current authentication.
- 2026-07-16 DYN-111: claim repair, source location/replacement plus hostile retrieval, and all mechanism repair attempts share their caller-owned 30/90/120-second end-to-end deadline. Capability/authentication probes and each curl subprocess consume the same remaining budget rather than resetting it.
- 2026-07-16 DYN-107: source verification uses HTTPS-only bounded curl retrieval and optional bounded PDF text extraction. One source replacement is permitted after a local check fails; a second failure is final. A separate non-browsing passage adjudicator can reject field mappings but cannot upgrade trust without exact fetched-byte coverage.
- 2026-07-16 DYN-112: the checked-in v1 breadth manifest has 266 unique explicit request texts across all 19 required categories, including 76 adversarial mutations and 25 explicit opt-in live-smoke selections spanning every category. Its metrics keep state, trust, identity, balance, evidence, mapping, presentation, latency, and failure dimensions separate.
- 2026-07-16 DYN-112: corpus scenarios intentionally record `oracle_reviewed_by: null` until independent chemistry review occurs. The harness reports those cases as unreviewed, so generated breadth cannot be represented as factual release accuracy. Live provider latency, three-platform packaging, and the packaged macOS journey remain explicit release evidence rather than unit-test claims.
- 2026-07-16 DYN-105/DYN-108: provider product names may drift from reviewed aliases, so product compilation now falls back to a unique reviewed formula or formula-bound external hint. Ambiguous and absent products remain formula-only and cannot acquire an invented structural graph.
- 2026-07-16 DYN-105: reviewed records with identical formula, charge, phase, representation, and canonical structural graph are internal aliases, not learner-visible chemical ambiguity. Composer identity resolution selects their stable lowest ID; same-formula records with different graphs remain an explicit choice.
- 2026-07-16 DYN-110: the compiler contract version advances when product identity recovery and static-capability diagnostics change, so cache replay recompiles under the new semantics instead of resurfacing a stale generic presentation result.
- 2026-07-16 DYN-111: provider commands run in isolated process groups and deadline/cancellation terminates the complete process tree. Source claims are capped at the smallest sufficient set of four so descendant-held pipes and sequential retrieval cannot silently extend the caller-owned bound.
- 2026-07-16 DYN-109: pinned Iced 0.14 does not expose an accessibility semantics API or macOS AccessKit integration. The app therefore adds keyboard-complete builder actions and exposes the current reactants/result capability in the native window title for automation and assistive inspection; a full control-level accessibility tree remains an upstream framework boundary rather than a fabricated local claim.
- 2026-07-16 DYN-108 (parity repair): family matching was inverted. It previously required every product to carry a resolved reviewed structure before any rule was tried, so a formula-only product silently blocked applicable families. Matching now binds only reactant structures, derives the products from each family's own reviewed selectors (`derive_generalized_products`), and compares derived products against the balanced declaration by exact element inventory and coefficient.
- 2026-07-16 DYN-108 (parity repair): reviewed aliases of one species can differ only in internal atom labels (for example `SodiumChloride` vs `NaFixedCation1Chloride`), which defeated label-sensitive graph equality during family matching. `ValidatedCatalogueBundle::structures_isomorphic` now proves exact label-independent equivalence with the bounded automorphism search machinery; constitutional isomers remain distinct, and a hit work limit fails closed to the resolver's own pick.
- 2026-07-16 DYN-108 (parity repair): products absent from the reviewed structure library no longer end escalation with a non-retryable static card. A closed structure-escalation contract asks the model for one graph per missing species; each proposal is compiled into an isolated `Working` catalogue bundle (trusted document plus the proposals under one provisional premise anchored to the internal explanatory-model evidence source) and crosses the identical catalogue validation as reviewed structures. Adopted products keep their species ids, cap identity confidence at `ExternalUnverified`, and carry explicit model-proposal provenance. Structure and mechanism escalation share one 120-second provider deadline per presentation generation.
- 2026-07-16 DYN-109 (parity repair): the "retry available" static state previously had no control. `RetryDynamicPresentation` relaunches only presentation enrichment on the retained validated static outcome; it never re-runs the claim, and a non-retryable static state cannot relaunch.
- 2026-07-16 DYN-110 (parity repair): cache v3 escalated recipes now persist the adopted structure proposal beside the mechanism response; offline replay re-adopts it through the identical isolated-bundle validation. Old escalated entries without structures still decode.
- 2026-07-16 DYN-112 (parity repair): the checked-in corpus expected plain `static` presentations as success for 50 cases, institutionalizing the missing parity requirement. Expectations are regenerated from the new `corpus-expectation-audit` binary, which resolves each scenario's reactants through the real registry and family matcher: 50 reviewed-family and 20 escalated-mechanism cases now expect full animation, no case expects `static` or `mechanism_unavailable_static` as success, and 107 cases honestly expect `invalid` where reactant identity (organics, bare ions, non-species tokens) is outside the current capability — the visible DYN-105 adapter roadmap. Three scenarios were repurposed onto resolvable chemistry to keep tier coverage: redox Zn + CuO, combustion Mg + CO2, and no-reaction Cu + H2O (gold has no reviewed metal structure).
- 2026-07-16 DYN-111 (live verification): live mechanism escalation had never actually run — the mechanism and structure output schemas used `oneOf`, tuple `prefixItems`/`items: false`, and bare `const`, all rejected by strict structured outputs with an error Codex emits on stdout JSONL and an empty stderr, which the invocation path then reported as a blank diagnostic. Both schemas are now strict-compatible (`anyOf`, homogeneous typed arrays, typed consts), a unit test pins the compatible subset, and `invoke` surfaces the last stdout error event whenever stderr is empty.
- 2026-07-16 DYN-108 (live verification): the live model systematically under-mapped coefficient instances and proposed electron states outside the reviewed valence domain. The escalation request now carries the exact `reactant_atom_paths`/`product_atom_paths` (every coefficient instance enumerated) and the reviewed `supported_states`/`metallic_states` for the involved elements, and the prompt states the instance-indexing and electrons-not-in-bonds conventions. With these, live Mg + CO2 produced a kernel-valid mechanism (12 frames) on the first attempt; the packaged app plays it in the standard 2D player and replays it offline from cache v3. `crates/agent/tests/live_probe.rs` remains as the explicit ignored live smoke for this path.
