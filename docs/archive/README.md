# Documentation archive

This directory preserves completed and superseded project records. It is an
audit trail, not current implementation authority.

- [`plans/`](plans/) — completed slice and migration plans;
- [`handoffs/`](handoffs/) — one-time agent or contributor contracts;
- [`catalogue/`](catalogue/) — completed catalogue execution, review, and
  coverage snapshots;
- [`contracts/`](contracts/) — superseded contracts whose durable rules now
  live in current architecture or verification documents;
- [`change-notes/`](change-notes/) — point-in-time compatibility notes; and
- [`qa/`](qa/) — completed visual and interaction evidence.

When an archived file disagrees with a durable source in `docs/`, the durable
source and current machine-readable artefacts win.
